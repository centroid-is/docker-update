Target shape (Plan 05-05):
```go
func NewServer(...) *Server {
    // Boot-time MIME registration — must run BEFORE any handler can fire
    mime.AddExtensionType(".js",   "application/javascript")
    mime.AddExtensionType(".css",  "text/css; charset=utf-8")
    mime.AddExtensionType(".svg",  "image/svg+xml")
    mime.AddExtensionType(".json", "application/json")
    mime.AddExtensionType(".woff2","font/woff2")
    // ...
}

func (s *Server) assetsHandler(w http.ResponseWriter, r *http.Request) {
    path := strings.TrimPrefix(r.URL.Path, "/assets/")
    f, err := s.assetsFS.Open(path)
    if err != nil {
        http.NotFound(w, r) // STRICT — no fallback to index.html
        return
    }
    defer f.Close()
    w.Header().Set("Cache-Control", "public, immutable, max-age=31536000")
    // MIME deduced by http.ServeContent / explicit set
    http.ServeContent(w, r, path, time.Time{}, f.(io.ReadSeeker))
}

func (s *Server) indexHandler(w http.ResponseWriter, r *http.Request) {
    w.Header().Set("Cache-Control", "no-cache")
    // serve dist/index.html
}

func (s *Server) stateHandler(w http.ResponseWriter, r *http.Request) {
    w.Header().Set("Cache-Control", "no-store")
    // existing memory-only serve
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Embedded FS (dist/) → HTTP | Plan 05-05 enforces strict 404 + immutable cache. A malicious or buggy client requesting `/assets/../../etc/passwd` is blocked by `assetsFS` (it's an `fs.Sub` of the embed; path traversal is impossible at the FS layer). |
| Browser cache → operator | `no-cache` on index.html + `no-store` on /api/state + `immutable` on /assets/* is the canonical SPA cache trio. Pitfall 8 reasonably closed. |
| Playwright clipboard read | Granted only to localhost test contexts; not a production concern. |
| Rebuild-binary helper | Runs `make ui`, `make build`, `docker compose ...` via execFile — argv discipline; no shell interpolation. |
| `mime.AddExtensionType` global state | Go's `mime` package is process-global. Tests that override would interfere if run in parallel — Phase 5 unit tests don't override (they only verify the registered values), so no flake. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05-01 | T (Tampering) | /assets/* path traversal | mitigate | `fs.Sub` of the embedded FS prevents `..` escape at the FS layer; http.NotFound on Open error catches anything anomalous. Phase 1 established this; Plan 05-05 preserves. |
| T-05-05-02 | I (Info Disclosure) | index.html fallback leak | mitigate | Strict 404 on /assets/* miss prevents the SPA from accidentally serving index.html bytes under a /assets/*.js URL (which is the actual Pitfall 8 — Chrome treats `text/html` under .js as a hard MIME error). Both unit test and e2e in-place-upgrade test assert. |
| T-05-05-03 | D (DoS) | infinite-cache poisoning | accept | `immutable max-age=31536000` means a tampered asset would be cached for 1 year — but the hash is content-derived; any tampering changes the URL, so the old hash either matches the bytes it shipped with or 404s. Acceptable. |
| T-05-05-04 | T | rebuild-binary helper subprocess | mitigate | execFile with argv; no shell. The only argv input is hardcoded strings — no user input flows in. |
| T-05-05-05 | I | /api/state caching | mitigate | `no-store` ensures the JSON never persists; in-place upgrades that change the wire shape can't serve stale JSON. |
| T-05-05-06 | T | Playwright test environment shared between specs | accept | Each spec uses `test.beforeEach` for navigation; clipboard state and toast state are per-context. The in-place-upgrade spec runs in serial mode (`.serial`) to avoid concurrent rebuilds. |
| T1 (phase-wide) | E | In-place upgrade asset poisoning | mitigate | This plan closes Pitfall 8 with both Go unit-test proof and end-to-end browser proof. The combination is the load-bearing invariant. |
| T2 (phase-wide) | A | Reduced-motion regression on spinner | n/a | Plan 05-01 handled the CSS exception; not Plan 05-05's surface. |
</threat_model>
