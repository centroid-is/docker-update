<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| UI → server | All POSTs go through `postAction`; URL is `encodeURIComponent`'d. The server (Phase 4) is the authority on safety labels, self-protection, mutex contention, and verify-after-recreate. UI is a UX accelerator. |
| Poll vs action race | `isActing` flag serializes by skipping polls during in-flight actions; the server's per-service mutex is the actual race-resolver. |
| Network failure | poll's catch swallows network errors silently; the header's "Xs ago" turning red is the only operator-facing signal (per UI-SPEC.md §10 — set by Header.svelte's derived color in Plan 05-02). |
| In-place upgrade | `cache: 'no-store'` on `/api/state` prevents stale JSON post-binary-swap. Plan 05-05 hardens `/assets/*` + index.html caching server-side. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04-01 | T (Tampering) | UI bypasses requiresWarning | accept | A user clicks Cancel then triggers another action via devtools — Server runs the action anyway; warning is operator-protective UX, not a security boundary. UI-SPEC.md and 05-CONTEXT.md acknowledge. |
| T-05-04-02 | T | Multiple rapid clicks on same row | mitigate | busyServices tracks per-row optimistic state; the server's per-service mutex (Phase 4 ACT-08) returns 409 on collision; UI surfaces a `Service busy` warning toast. Two-layer defense. |
| T-05-04-03 | D (DoS) | Poll loop leaks | mitigate | $effect cleanup clearInterval on App.svelte teardown (HMR safety; production single-mount). |
| T-05-04-04 | I (Info Disclosure) | ActionError.reason in toast | accept | Same exposure boundary as Phase 4 wire (T-04-01-02); LAN-only deployment. |
| T-05-04-05 | E (Elevation) | pollNow 404 fallback | mitigate | `pollNow()` returning false routes to a regular `poll()` — operator never loses the Watch-now affordance even if Phase 3 didn't ship `/api/poll-now`. Defensive against forward-compat drift. |
| T-05-04-06 | T | Stale state after in-place upgrade | mitigate | `cache: 'no-store'` on /api/state + Plan 05-05's index.html `no-cache` + /assets/* immutable. Three-layer defense; Pitfall 8 closed. |
| T-05-04-07 | D | Toast queue unbounded growth | mitigate | Non-error auto-dismiss 5s (Plan 05-03). Error stack acceptable per 05-03 disposition. |
| T-05-04-08 | T | encodeURIComponent bypass | mitigate | Server's ACT-10 allowlist regex is the authoritative gate (Phase 4); encodeURIComponent is defense in depth. |
| T1 (phase-wide) | E | Asset poisoning across in-place upgrade | partial-mitigate | Plan 05-04 lands the client-side `cache: 'no-store'`; Plan 05-05 hardens server headers. Together they close Pitfall 8. |
</threat_model>
