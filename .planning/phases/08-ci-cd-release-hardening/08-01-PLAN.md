<!-- Phase 1 ci.yml shape (CURRENT — three jobs: go, ui, e2e). Phase 8 rewrites in-place. -->

The Phase 1 baseline ci.yml has these jobs:
- `go`: checkout + setup-go@v5 + go vet + go test + tygo + make check-types
- `ui`: checkout + setup-node@v4 + npm ci + npm run build
- `e2e`: needs [go, ui]; checkout + setup-go + setup-node + install oras + install Playwright + make e2e + upload report on failure

Phase 8 keeps the JOB NAMES `go`, `ui`, `e2e` (so existing PR status checks don't break) but
factors out lint (golangci-lint) and tygo (already in go-job tail) into a dedicated `lint` job,
adds an `image-build` job between ui and e2e, and adds a `ghcr-smoke-pr` job parallel to e2e.

Final shape (this plan):

```
checkout → setup-go (1.26) ─┐
                            ├─ lint (go vet + golangci-lint + make check-types) ─┐
checkout → setup-node (22) ─┘                                                    │
                                                                                 ├─ image-build (docker build + size gate) ─┐
checkout → go (go test ./... -race)                                              │                                          │
checkout → ui (npm ci + npm run build) ──────────────────────────────────────────┘                                          │
                                                                                                                            ├─ e2e (make e2e-cron-fast)
                                                                                                                            │
checkout → ghcr-smoke-pr (crane digest ghcr.io/distroless/static-debian12:nonroot, no creds) ───────────────────────────────┘
```

Job dependency graph (in `needs:` terms):
- `lint`: needs nothing (parallel-startable)
- `go` (unit tests): needs `lint`
- `ui`: needs `lint`
- `image-build`: needs [`go`, `ui`]
- `e2e`: needs [`image-build`]
- `ghcr-smoke-pr`: needs nothing (parallel-startable; no creds in scope)

The chain stops at any red node — GitHub Actions' `needs:` graph is fail-fast by default.

A future Phase 8 plan (08-02) will hand off to `publish.yml` via `workflow_run` listening for
the `ci` workflow to complete successfully on `main`.
</interfaces>

    ```make
    # Image-size budget gate. Matches the Phase 8 CI step byte-for-byte so a developer
    # can run the same gate locally before pushing. The default IMAGE is hmi-update:dev
    # so `make image && make image-size-check` works without an env var. CI overrides
    # with IMAGE=hmi-update:ci. The budget defaults to 30,000,000 bytes (30 MB N6
    # constraint); HMI_UPDATE_CI_IMAGE_SIZE_BUDGET overrides for future arm64 / debug
    # builds (CONTEXT.md Configuration Knobs).
    IMAGE ?= hmi-update:dev
    HMI_UPDATE_CI_IMAGE_SIZE_BUDGET ?= 30000000

    image-size-check:
    	@SIZE=$$(docker image inspect --format '{{.Size}}' $(IMAGE)) ; \
    	  echo "[image-size-check] image=$(IMAGE) size=$$SIZE bytes budget=$(HMI_UPDATE_CI_IMAGE_SIZE_BUDGET) bytes" ; \
    	  if [ "$$SIZE" -gt "$(HMI_UPDATE_CI_IMAGE_SIZE_BUDGET)" ]; then \
    	    echo "FAIL: image $(IMAGE) is $$SIZE bytes (over $(HMI_UPDATE_CI_IMAGE_SIZE_BUDGET) byte budget)" >&2 ; \
    	    exit 1 ; \
    	  fi ; \
    	  echo "OK: image $(IMAGE) within budget"
    ```
