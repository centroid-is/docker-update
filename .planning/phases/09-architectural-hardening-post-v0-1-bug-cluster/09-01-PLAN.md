  <action>
    APPEND a new target to `Makefile` (do not modify existing targets):

    ```
    .PHONY: grep-no-compose
    grep-no-compose:
    	@if grep -rE 'exec\.Command|docker compose|compose up' internal/actions/ internal/recreate/ 2>/dev/null | grep -v '_test\.go' | grep -v '^[^:]*:[[:space:]]*//' ; then \
    		echo 'FAIL: subprocess-compose pattern detected in non-test production code (SC-1)'; \
    		exit 1; \
    	else \
    		echo 'PASS: grep-no-compose'; \
    	fi
    ```

    Notes on the grep design (per the planner's grep-gate-hygiene rule):
      - Searches `internal/actions/` AND `internal/recreate/`. `internal/recreate/` does not exist yet (Plan 09-03 creates it); `grep -r` against a missing directory emits a warning to stderr but the `2>/dev/null` swallows it and the recursive grep against the existing directory still works. After Plan 09-03 lands, the new directory will be included automatically.
      - Excludes `_test.go` files by line content (`grep -v '_test\.go'` matches the filename in `grep -r`'s `path:line:text` output).
      - Excludes Go single-line comments (`grep -v '^[^:]*:[[:space:]]*//'`) so historical commentary mentioning `docker compose` does not trip the gate.
      - The `if grep ... ; then FAIL` shape is the standard "non-zero exit on match" idiom; the explicit `echo FAIL: ...` line identifies the gate by SC.

    Then verify `internal/api/dist/.gitkeep` is tracked in git (per RESEARCH.md Pitfall 6, the file currently exists and is tracked; the defensive `mkdir -p` in Task 1 covers the fresh-clone edge case). If `git ls-files internal/api/dist/.gitkeep` returns nothing, create an empty file at that path and `git add` it. Otherwise no-op.
