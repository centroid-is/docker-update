<objective>
Ship the `internal/docker.Discoverer` — the single-consumer goroutine that:
  1. At boot, calls `Client.ContainerList` with the `hmi-update.watch=true` label filter and seeds `state.Containers`.
  2. Subscribes to Docker events filtered to `type=container`, `event=start|die|destroy`; for each event, inspects the container (when needed), maps to a state mutation, and calls `state.Store.Update`.
  3. On events-stream disconnect, reconnects with exponential backoff (1s → 2s → 4s → ... → 30s cap) and re-runs the boot list on every successful reconnect.

Purpose: DOCK-04 — "Containers labeled hmi-update.watch=true enumerated and visible in /api/state within 60 s of docker compose up -d." This plan owns the daemon-side mechanism. It is the first long-running goroutine in the codebase, and it establishes the single-consumer convention from ARCHITECTURE.md that Phase 3's cron poller will join.

Output: A `Discoverer` struct returned by `NewDiscoverer(client docker.Client, store *state.Store)`, a blocking `Run(ctx) error` method that the main goroutine calls in a separate goroutine, and a unit test that drives a FakeClient through the full lifecycle (boot, start, die, destroy, reconnect) without touching a real daemon.

This plan depends on plan 02-01 (the docker facade interface). It is the only plan in Wave 2 — plan 02-02 (compose reader) ran in parallel with 02-01 in Wave 1.

    1. Create `internal/docker/discovery.go`. The file's package doc + struct + constructor + Run method. Key structural decisions, all documented inline:
       - `NewDiscoverer(client docker.Client, store *state.Store) *Discoverer` returns a handle; Phase 4 will add `Stop()` for graceful shutdown.
       - `Run(ctx context.Context) error` blocks until ctx is cancelled OR a non-recoverable error occurs (none in v1 — Events errors are recoverable via backoff). Returns ctx.Err() on cancellation.
       - The boot ContainerList runs synchronously at the start of Run, BEFORE the event loop. Phase 4 will care about whether boot completed before HTTP requests arrive; plan 02-04's main.go wiring spawns Run in a goroutine and lets the HTTP server come up immediately, so an early `/api/state` poll may return an empty `containers` map. This is documented in CONTEXT.md `<specifics>`.
       - The reconnect backoff is `func() time.Duration` that returns 1s on the first failure, 2s on the second, doubling up to a 30s cap. Track attempt count in a local int.
       - The sleeper is a `func(d time.Duration)` field on Discoverer, defaulting to `time.Sleep`, so tests can substitute a no-op or controllable sleeper.

       Skeleton:
       ```go
       // Package docker (continued). discovery.go owns the single producer
       // goroutine that turns Docker daemon events into state.Store mutations.
       package docker

       import (
           "context"
           "errors"
           "fmt"
           "log/slog"
           "strings"
           "time"

           "github.com/centroid-is/hmi-update/internal/state"
       )

       // Discoverer is the single producer of container-related state
       // mutations in Phase 2. It owns one boot ContainerList call and one
       // long-running Events subscription with reconnect logic.
       //
       // Phase 3's cron poller will be the second producer (registry digest
       // checks); both producers feed state through the same state.Store.Update
       // surface — see ARCHITECTURE.md Pattern 3 "single-consumer channel for
       // state mutations." Today there is no channel because there is only one
       // producer; the channel materializes in Phase 3 if/when the
       // poller-vs-discovery interleaving needs a serialization seam.
       //
       // Anti-deadlock invariant (ARCHITECTURE.md lines 419-420):
       //   Discoverer NEVER calls dockerClient.ContainerInspect from inside
       //   state.Store.Update's closure. Inspect FIRST, then call Update with
       //   the resolved fields. The closure is a pure map-mutation function.
       type Discoverer struct {
           // client is the package-local Client interface from client.go
           // (plan 02-01). Declaring it as the bare type name is fine because
           // Discoverer lives in the same package — no alias needed.
           client Client
           store  *state.Store

           // sleeper is the back-off sleep function. Default is time.Sleep;
           // tests substitute a no-op so the reconnect-backoff test runs
           // without actually sleeping 1+2+4+8 = 15 seconds.
           sleeper func(time.Duration)

           // maxBackoff caps the exponential progression. CONTEXT.md
           // <specifics> says "up to 30s." A future operator who finds the
           // daemon is flapping every 35s might raise this; expose as a
           // field to keep the test deterministic.
           maxBackoff time.Duration

           // backoffBase is the first sleep after the first failure.
           // CONTEXT.md says 1s.
           backoffBase time.Duration
       }

       // NewDiscoverer constructs a Discoverer wired to the supplied client
       // and store. Default sleeper is time.Sleep; default backoff is
       // 1s base, 30s cap (per CONTEXT.md <specifics>).
       func NewDiscoverer(client Client, store *state.Store) *Discoverer {
           return &Discoverer{
               client:      client,
               store:       store,
               sleeper:     time.Sleep,
               maxBackoff:  30 * time.Second,
               backoffBase: 1 * time.Second,
           }
       }

       // Run executes the boot ContainerList once, then enters the events
       // subscription loop with reconnect. Run returns when ctx is cancelled.
       // Errors from individual events are logged and recovered; only ctx
       // cancellation terminates Run.
       func (d *Discoverer) Run(ctx context.Context) error {
           slog.Info("discovery.boot.start", "label_filter", "hmi-update.watch=true")
           if err := d.bootList(ctx); err != nil {
               // Log but do not exit — the event loop may still produce
               // valid state mutations once the daemon comes back. The
               // /healthz handler is the authoritative liveness signal.
               slog.Error("discovery.boot.fail", "err", err)
           }
           return d.eventsLoop(ctx)
       }

       // bootList runs a single ContainerList with the hmi-update.watch=true
       // filter and seeds state.Containers. Each container is inspected via
       // a synthetic upsert that mirrors the start-event handler — so a
       // container that appears in boot AND in an event burst is handled
       // identically.
       func (d *Discoverer) bootList(ctx context.Context) error {
           opts := ContainerListOptions{
               // The exact field name depends on the moby SDK signature
               // discovered in plan 02-01. If the SDK takes a Filters builder,
               // use it; otherwise use the relevant equivalent. Document the
               // discovered shape inline.
               // Filters: Filters{}.Add("label", "hmi-update.watch=true"),
           }
           containers, err := d.client.ContainerList(ctx, opts)
           if err != nil {
               return fmt.Errorf("ContainerList: %w", err)
           }
           slog.Info("discovery.boot.list", "count", len(containers))
           for _, c := range containers {
               d.upsertFromInspect(ctx, c.ID /* or whatever field name the SDK uses */)
           }
           return nil
       }

       // eventsLoop subscribes to docker events; on stream error or daemon
       // disconnect, reconnects with exponential backoff capped at 30s.
       // After a successful reconnect, re-runs the boot list (per CONTEXT.md
       // <specifics> "After a successful reconnect, re-run the boot
       // ContainerList to catch any state changes that occurred while
       // disconnected").
       func (d *Discoverer) eventsLoop(ctx context.Context) error {
           attempt := 0
           for {
               if err := ctx.Err(); err != nil {
                   return err
               }

               opts := EventsListOptions{
                   // Same SDK-shape caveat as bootList; filter to type=container
                   // and event=start|die|destroy. The exact filter syntax
                   // depends on the SDK's Filters builder.
                   // Filters: Filters{}.
                   //     Add("type", "container").
                   //     Add("event", "start").
                   //     Add("event", "die").
                   //     Add("event", "destroy"),
               }
               eventCh, errCh := d.client.Events(ctx, opts)

               // Reset attempt on every successful subscription. We define
               // "successful" as: the subscription returned without an
               // immediate error AND we received at least one event OR ctx
               // is still live. The simpler approximation here is "reset on
               // subscription return"; if Events errors instantly we'll bump
               // the attempt in the error path.
               if attempt > 0 {
                   slog.Info("discovery.events.reconnected", "attempt", attempt)
                   // Re-run boot list to catch missed changes during the gap.
                   if err := d.bootList(ctx); err != nil {
                       slog.Error("discovery.reconnect.boot.fail", "err", err)
                   }
               }
               attempt = 0

               // Drain events until eventCh or errCh closes, or ctx done.
               drained := d.drainEvents(ctx, eventCh, errCh)

               if err := ctx.Err(); err != nil {
                   return err
               }

               attempt++
               backoff := d.computeBackoff(attempt)
               slog.Warn("discovery.events.reconnect",
                   "attempt", attempt,
                   "backoff_ms", backoff.Milliseconds(),
                   "drain_reason", drained)
               d.sleeper(backoff)
           }
       }

       // computeBackoff returns 1s * 2^(attempt-1), capped at maxBackoff.
       // attempt is 1-indexed. The first failure sleeps backoffBase.
       func (d *Discoverer) computeBackoff(attempt int) time.Duration {
           if attempt < 1 {
               attempt = 1
           }
           // Use bit shift carefully to avoid overflow for absurd attempt counts.
           // After ~30 attempts a uint64 of nanoseconds overflows; cap before then.
           const safeMaxAttempt = 30
           if attempt > safeMaxAttempt {
               return d.maxBackoff
           }
           b := d.backoffBase << (attempt - 1)
           if b > d.maxBackoff || b <= 0 {
               return d.maxBackoff
           }
           return b
       }

       // drainEvents consumes events until the stream closes or errors.
       // Returns a string describing why the drain ended for slog.
       func (d *Discoverer) drainEvents(ctx context.Context, eventCh <-chan EventMessage, errCh <-chan error) string {
           for {
               select {
               case <-ctx.Done():
                   return "ctx-cancelled"
               case err, ok := <-errCh:
                   if !ok {
                       return "errch-closed"
                   }
                   if err != nil {
                       slog.Warn("discovery.events.stream.err", "err", err)
                       return "stream-err"
                   }
               case ev, ok := <-eventCh:
                   if !ok {
                       return "eventch-closed"
                   }
                   d.handleEvent(ctx, ev)
               }
           }
       }

       // handleEvent dispatches a single event message. The exact field name
       // on EventMessage (Action vs Status; Actor.ID vs ID) depends on the
       // moby SDK shape — see plan 02-01's Client interface for the
       // discovered shape. If the SDK uses a nested Actor struct (it does in
       // legacy docker/docker), use ev.Actor.ID and ev.Action.
       func (d *Discoverer) handleEvent(ctx context.Context, ev EventMessage) {
           action := /* extract from ev — e.g. ev.Action or ev.Status */ ""
           id := /* extract container id — e.g. ev.Actor.ID or ev.ID */ ""
           slog.Info("discovery.event.received", "action", action, "container_id", shortID(id))
           switch action {
           case "start":
               d.upsertFromInspect(ctx, id)
           case "die":
               d.markStopped(ctx, id)
           case "destroy":
               d.removeContainer(ctx, id)
           default:
               // Unhandled (we filter to start/die/destroy at the SDK level;
               // this default is defensive in case the SDK delivers extra).
           }
       }

       // upsertFromInspect calls ContainerInspect THEN state.Store.Update.
       // The inspect happens OUTSIDE the store lock per the anti-deadlock
       // rule (ARCHITECTURE.md lines 419-420). DO NOT inline the inspect
       // into the Update closure — that closure runs under state.Store.mu
       // and any blocking call inside it stalls every reader.
       func (d *Discoverer) upsertFromInspect(ctx context.Context, id string) {
           if id == "" {
               return
           }
           insp, err := d.client.ContainerInspect(ctx, id)
           if err != nil {
               slog.Error("discovery.inspect.fail", "container_id", shortID(id), "err", err)
               return
           }
           svc := serviceLabel(insp)
           if svc == "" {
               // No compose service label -> not a watched-via-compose container; skip.
               return
           }
           img, tag := parseImageRef(/* insp.Config.Image — exact field path per SDK */ "")
           pinned := strings.Contains(/* insp.Config.Image */ "", "@sha256:")
           filteredLabels := filterHmiLabels(/* insp.Config.Labels — map[string]string */ nil)

           if err := d.store.Update(func(st *state.State) {
               c := st.Containers[svc]
               c.Service = svc
               c.Image = img
               c.Tag = tag
               c.ContainerID = shortID(id)
               c.Labels = filteredLabels
               c.Pinned = pinned
               c.Stopped = false // we just saw a start event; clear any prior die marker
               st.Containers[svc] = c
           }); err != nil {
               slog.Error("discovery.event.start.persist", "service", svc, "err", err)
           }
       }

       // markStopped sets Container.Stopped = true while preserving every
       // other field. Phase 5's status badge consumes this; nothing in
       // Phase 2 changes behaviour for stopped rows.
       func (d *Discoverer) markStopped(ctx context.Context, id string) {
           svc := d.serviceForContainerID(ctx, id)
           if svc == "" {
               return
           }
           if err := d.store.Update(func(st *state.State) {
               c, ok := st.Containers[svc]
               if !ok {
                   return
               }
               c.Stopped = true
               st.Containers[svc] = c
           }); err != nil {
               slog.Error("discovery.event.die.persist", "service", svc, "err", err)
           }
       }

       // removeContainer deletes the row entirely. Phase 5's UI will not see
       // a "destroyed" badge; if the container reappears under the same
       // service name later (compose recreate), a fresh start event will
       // bring it back.
       func (d *Discoverer) removeContainer(ctx context.Context, id string) {
           svc := d.serviceForContainerID(ctx, id)
           if svc == "" {
               return
           }
           if err := d.store.Update(func(st *state.State) {
               delete(st.Containers, svc)
           }); err != nil {
               slog.Error("discovery.event.destroy.persist", "service", svc, "err", err)
           }
       }

       // serviceForContainerID looks up the service name in the existing
       // state map by matching ContainerID. For die/destroy events we don't
       // need to re-inspect — the row already has the service name.
       func (d *Discoverer) serviceForContainerID(_ context.Context, id string) string {
           short := shortID(id)
           snapshot := d.store.Get()
           for svc, c := range snapshot.Containers {
               if c.ContainerID == short {
                   return svc
               }
           }
           return ""
       }

       // shortID returns the leading 12 characters of id (the standard
       ```

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Tampering | Container labels populated into state.Container.Labels | mitigate | filterHmiLabels accepts ONLY keys with the `hmi-update.` prefix. A container that crafts arbitrary labels cannot pollute the state map with unrelated keys (e.g. could not inject `is_admin=true`). |
| T-02-03-02 | Denial of service | Event-stream burst (e.g. a noisy compose restart) | mitigate | The events channel is buffered (size 16 in plan 02-01). For sustained bursts, the SDK's stream blocks the daemon, which is acceptable — the daemon is not the attacker. Discoverer processes one event at a time; if the buffer fills, the upstream goroutine blocks until handleEvent drains. |
