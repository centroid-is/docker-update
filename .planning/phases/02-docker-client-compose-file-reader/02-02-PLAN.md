<objective>
Ship the `internal/compose.Reader` package — a stat-based, no-YAML compose-file drift detector that protects every Phase 4 mutating action against Pitfall 10 (a compose file replaced under the bind-mount by an atomic-save editor or operator relocation).

Purpose: DOCK-02 requires "stat-before-act and inode-drift detection." This is the cheapest possible primitive that satisfies the requirement: a one-time stat at boot, an O(1) re-stat per call. Phase 4's update/rollback handlers will invoke `CheckUnchanged` immediately before `docker compose -f $HMI_UPDATE_COMPOSE_PATH up -d --force-recreate <svc>`; if drift is detected, the action errors with HTTP 412 and a clear remediation hint instead of acting on a stale or destroyed bind-mount.

Output: A `Reader` struct, an `ErrComposeFileMoved` sentinel, and a paired test file that proves the atomic-rename drift signal is detected, the in-place edit drift signal (mtime change) is detected, and the no-op case stays cheap and correct.

This plan is independent of plan 02-01 (docker facade) and plan 02-03 (discovery goroutine); it runs parallel to plan 02-01 in Wave 1 (no file overlap — different packages) and precedes plan 02-03 in Wave 2.
</objective>

    2. Create `internal/compose/reader.go` with the Reader struct + NewReader + CheckUnchanged. Mirror the constructor structure of `internal/state/store.go` lines 22-93. The struct:
       ```go
       package compose

       import (
           "fmt"
           "log/slog"
           "os"
           "sync"
           "syscall"
           "time"
       )

       // Reader holds the boot snapshot of the compose file's stat metadata
       // and exposes a CheckUnchanged method that re-stats the path on demand.
       //
       // Reader is safe for concurrent use. CheckUnchanged is read-only and
       // may be called from any goroutine; the RWMutex protects the boot
       // snapshot in the (currently impossible) case where a future
       // CONTEXT.md decision rotates the snapshot at runtime. The mutex is
       // taken in read mode for CheckUnchanged; no caller in v1 takes write
       // mode after construction.
       type Reader struct {
           path string

           mu sync.RWMutex
           // bootInode is set at construction. On Linux, syscall.Stat_t.Ino
           // gives a stable inode that uniquely identifies the file at that
           // moment. If a future operator atomic-saves the compose file, the
           // new file has a different inode and CheckUnchanged trips
           // ErrComposeFileMoved.
           bootInode uint64
           // bootModTime + bootSize are the fallback signal for filesystems
           // where inodes are not stable (some FUSE mounts return inode 0 or
           // reuse inodes within a session). They are also a belt-and-braces
           // signal for in-place edits: same inode but updated content.
           bootModTime time.Time
           bootSize    int64
           // bootInodeStable is false when the filesystem returned inode 0
           // (or the OS doesn't expose syscall.Stat_t at all). In that case
           // CheckUnchanged compares (mtime, size) only.
           bootInodeStable bool
       }

       // NewReader stats the compose file at path and captures a boot
       // snapshot. NewReader fails fast on:
       //   - empty path (an unset HMI_UPDATE_COMPOSE_PATH env var)
       //   - the file not existing or not being stattable
       //
       // The caller (cmd/hmi-update/main.go in plan 02-04) wraps the error
       // and calls log.Fatalf so the operator sees the cause at boot.
       func NewReader(path string) (*Reader, error) {
           if path == "" {
               return nil, fmt.Errorf("compose.NewReader: empty path (set HMI_UPDATE_COMPOSE_PATH)")
           }
           r := &Reader{path: path}
           if err := r.captureBootSnapshot(); err != nil {
               return nil, fmt.Errorf("compose.NewReader: %w", err)
           }
           return r, nil
       }

       func (r *Reader) captureBootSnapshot() error {
           info, err := os.Stat(r.path)
           if err != nil {
               return err
           }
           r.mu.Lock()
           defer r.mu.Unlock()
           r.bootModTime = info.ModTime()
           r.bootSize = info.Size()

           if st, ok := info.Sys().(*syscall.Stat_t); ok && st.Ino != 0 {
               r.bootInode = uint64(st.Ino)
               r.bootInodeStable = true
               slog.Info("compose.reader.boot",
                   "path", r.path,
                   "inode", r.bootInode,
                   "mtime", r.bootModTime.Format(time.RFC3339Nano),
                   "size", r.bootSize,
                   "drift_signal", "inode-primary")
           } else {
               // Fallback: (mtime, size) only. Document in slog so a future
               // operator on a FUSE/NFS HMI knows why the signal is weaker.
               r.bootInodeStable = false
               slog.Info("compose.reader.boot",
                   "path", r.path,
                   "mtime", r.bootModTime.Format(time.RFC3339Nano),
                   "size", r.bootSize,
                   "drift_signal", "mtime-size-fallback")
           }
           return nil
       }

       // CheckUnchanged re-stats the compose file and returns:
       //   - nil if the file's inode (when stable) AND (mtime, size) match
       //     the boot snapshot.
       //   - a wrapped ErrComposeFileMoved if any of those signals differ.
       //   - a wrapped fs.ErrNotExist (or other os error) if the stat itself
       //     fails. Phase 4 callers treat fs.ErrNotExist as a special case
       //     of "moved" — the file was deleted. We unify both under
       //     ErrComposeFileMoved because the operator remediation is the
       //     same: restart hmi-update.
       //
       // CheckUnchanged is O(1) — a single stat syscall — so calling it
       // before every action is cheap even on the busiest HMIs (10s of
       // actions per minute would be exotic).
       func (r *Reader) CheckUnchanged(ctx context.Context) error {
           // ctx is accepted for API symmetry and future cancellation
           // (e.g. if a phase wants to stat with a timeout); the current
           // implementation does not block, so ctx is unused. Document.
           _ = ctx

           info, err := os.Stat(r.path)
           if err != nil {
               // File deleted or otherwise unreachable -> treat as drift.
               // Wrap so errors.Is(err, ErrComposeFileMoved) is true AND
               // errors.Is(err, fs.ErrNotExist) is true (preserving both
               // signals for callers that want to distinguish).
               return fmt.Errorf("compose.CheckUnchanged stat %s: %w: %w", r.path, ErrComposeFileMoved, err)
           }

           r.mu.RLock()
           defer r.mu.RUnlock()

           // Compare (mtime, size) — always; this is the belt-and-braces
           // signal that also catches in-place edits.
           if !info.ModTime().Equal(r.bootModTime) || info.Size() != r.bootSize {
               return fmt.Errorf("compose.CheckUnchanged %s: mtime/size drift (boot mtime=%s size=%d, now mtime=%s size=%d): %w",
                   r.path,
                   r.bootModTime.Format(time.RFC3339Nano), r.bootSize,
                   info.ModTime().Format(time.RFC3339Nano), info.Size(),
                   ErrComposeFileMoved)
           }

           // Compare inode (when stable).
           if r.bootInodeStable {
               if st, ok := info.Sys().(*syscall.Stat_t); ok {
                   if uint64(st.Ino) != r.bootInode {
                       return fmt.Errorf("compose.CheckUnchanged %s: inode drift (boot=%d, now=%d): %w",
                           r.path, r.bootInode, uint64(st.Ino), ErrComposeFileMoved)
                   }
               }
           }

           return nil
       }
       ```
       NOTE on `syscall.Stat_t`: this is Linux/Darwin specific. The hmi-update binary targets `linux/amd64` only in v1 per CLAUDE.md, but `go test` runs on both Linux and Darwin developer machines and CI runners. `syscall.Stat_t` exists on both Linux and Darwin with the `Ino` field, so the code compiles. On Windows the field doesn't exist — out of scope.

       Add the `"context"` import to reader.go for `CheckUnchanged(ctx context.Context)`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Process → Host filesystem | The Reader stats a host file mounted via bind-mount into the container. A symlink swap or directory-mount swap on the host could redirect the stat to an unexpected path. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Tampering | The compose file pointed at by HMI_UPDATE_COMPOSE_PATH | mitigate | Reader detects ANY drift signal (inode OR mtime OR size) and returns ErrComposeFileMoved. Phase 4 maps this to HTTP 412 — the operator sees a clear error rather than the action running against an unexpected file. |
| T-02-02-02 | Spoofing | Symlink swap on the host pointing the bind-mount at a different file | accept | A determined attacker with host root can do worse things; the Reader's inode comparison still detects the swap because the resolved file's inode changes. Accept residual risk. |
| T-02-02-03 | Information disclosure | CheckUnchanged error messages echo the file path and stat fields | mitigate | The path comes from the operator-controlled HMI_UPDATE_COMPOSE_PATH env var, not from an attacker-controlled API parameter; echoing it back in error messages is intentional remediation guidance. Bigger risk would be echoing socket paths in /healthz responses — that lives in plan 02-04's threat register, not here. |
| T-02-02-04 | Denial of service | An attacker who can change the compose file mtime constantly forces every action to 412 | accept | The remediation is `restart hmi-update`. An attacker with host-side write access to the compose file already controls the HMI. Out of scope. |
</threat_model>
