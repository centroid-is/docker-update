# Phase 1 — UI Design Contract

> Visual and interaction contract for the empty Svelte 5 + Vite + Tailwind v4 shell that ships in Phase 1. **The real operator UI lives in Phase 5.** This contract intentionally locks only the design tokens, layout primitives, typography, color baseline, and empty-state surface so Phase 5 inherits a coherent system. Generated by `gsd-ui-researcher`, verified by `gsd-ui-checker`.

**Audience:** Centroid field engineers (internal technical operators), 1024 px+ desktop browser on the HMI LAN. No mobile, no public-facing accessibility hardening beyond keyboard-operable buttons and visible focus rings (relevant to Phase 5; Phase 1 has no interactive controls).

**Phase 1 UI surface (what actually ships):**
1. Page shell `ui/src/App.svelte` — single page at `/`, no SPA router.
2. Header bar: app name `hmi-update` left-aligned. Optional build/version info right-aligned (may be deferred to Phase 5).
3. A single max-width page container.
4. Table component `ui/src/lib/Table.svelte` with `Props = { containers: Container[] }`, six column headers, and an empty-state row when `containers` is empty.
5. Generated TS types (`ui/src/lib/types.d.ts`) imported and used as Props types — proves the tygo pipeline end-to-end.

**Source pre-population:** Most fields below are pre-populated from `01-CONTEXT.md` and `research/STACK.md`. The user instructed autonomous operation; nothing in this contract was asked back to the user.

## Spacing Scale

Tailwind v4 ships an 8-point–compatible spacing scale rooted at a 4 px base. Declared values (every value is a multiple of 4):

| Token | Tailwind class | Value | Phase 1 Usage |
|-------|---------------|-------|---------------|
| xs | `p-1` / `gap-1` | 4 px | Reserved (icon gaps in Phase 5) |
| sm | `p-2` / `gap-2` | 8 px | Table cell padding (vertical) |
| md | `p-4` / `gap-4` | 16 px | Table cell padding (horizontal); header bar vertical padding |
| lg | `p-6` / `gap-6` | 24 px | Header bar horizontal padding; page container top/bottom padding |
| xl | `p-8` / `gap-8` | 32 px | Empty-state row vertical padding (visual breathing room) |
| 2xl | `p-12` / `gap-12` | 48 px | Reserved (Phase 5 major section breaks) |
| 3xl | `p-16` / `gap-16` | 64 px | Reserved (Phase 5 page-level spacing) |

**Page container:** `max-w-screen-xl mx-auto px-6` (max-width 1280 px, 24 px horizontal gutter). Fits the 1024 px+ HMI desktop target with room to grow when Phase 5 adds per-row action buttons.

**Exceptions:** none. All declared values are Tailwind v4 defaults (multiples of 4 px). No bespoke pixel values are permitted; Phase 5 must extend this scale rather than introduce arbitrary values.

## Typography

Phase 1 declares **4 sizes** and **2 weights**. Phase 5 inherits all four sizes; semantic role mapping (e.g. status-badge font size) is Phase 5's to assign.

| Role | Tailwind class | Size | Weight | Line Height | Usage |
|------|----------------|------|--------|-------------|-------|
| Display | `text-2xl font-semibold` | 24 px | 600 | 1.2 (`leading-tight`) | App name `hmi-update` in the header bar |
| Heading | `text-lg font-semibold` | 18 px | 600 | 1.3 (`leading-snug`) | Reserved for Phase 5 section headings; not used in Phase 1 |
| Body | `text-sm font-normal` | 14 px | 400 | 1.5 (`leading-relaxed`) | Table headers, empty-state copy, header build-info |
| Label / caption | `text-xs font-normal` | 12 px | 400 | 1.4 (`leading-normal`) | Reserved for Phase 5 status badges and meta info |

**Monospace (digest cells):**

| Role | Tailwind class | Size | Weight | Line Height | Usage |
|------|----------------|------|--------|-------------|-------|
| Digest (mono) | `font-mono text-xs` | 12 px | 400 | 1.4 | Reserved — `current digest` / `available digest` / `previous digest` columns in Phase 5. Phase 1 columns are header-only (empty body), so no digest text renders, but the contract locks the typeface so Phase 5 can't drift. |

**Weights used in Phase 1:** exactly two — `font-normal` (400) and `font-semibold` (600). No `font-medium` (500), no `font-bold` (700). Phase 5 must not introduce additional weights without amending this contract.

**Headings line-height policy:** 1.2 for display, 1.3 for heading. Body and label use 1.4–1.5 to maximize scannability of long-form copy (empty-state body, future toast messages in Phase 5).

## Color

Phase 1 follows a strict 60 / 30 / 10 split using Tailwind v4 `zinc` ramp tokens. Semantic state colors (status-badge palette for `up-to-date` / `update-available` / `rollback-available` / `disabled`) are **deferred to Phase 5** — slot reserved below.

| Role | Tailwind class | Value (light mode) | Usage |
|------|---------------|---------------------|-------|
| **Dominant (60%)** | `bg-zinc-50` | `#fafafa` | Page background, table body surface, empty-state row background |
| **Secondary (30%)** | `bg-white` (cards) and `bg-zinc-100` (table header row, header bar) | `#ffffff` / `#f4f4f5` | Header bar background, table header row, future card containers in Phase 5 |
| **Accent (10%)** | `text-zinc-900` (primary text + borders via `border-zinc-200`) | `#18181b` text, `#e4e4e7` borders | Body text, table header text, column-divider borders, focus-ring base color |
| **Destructive** | `text-red-600` / `bg-red-50` | `#dc2626` text, `#fef2f2` surface | **Reserved for Phase 5** — pre-action "display may flicker" warning toasts on `flutter`/`weston` Update (UI-08); destructive-action confirmation dialogs in Phase 4 |

**Accent reserved for:** body text rendered against the dominant surface; the 1-pixel column dividers between the six table-header cells; the visible keyboard focus ring (`focus-visible:ring-2 focus-visible:ring-zinc-900 focus-visible:ring-offset-2`) on any focusable element introduced in Phase 5. **Never used** as a fill color for buttons or rows in Phase 1.

**Deferred semantic palette (Phase 5 must declare):**

| Status (Phase 5) | Reserved Tailwind hue | Rationale |
|------------------|------------------------|-----------|
| `up-to-date` | `green-600` text on `green-50` surface | Calm success — high contrast on white, low retinal load |
| `update-available` | `amber-600` text on `amber-50` surface | Attention without alarm — distinguishes from destructive red |
| `rollback-available` | `blue-600` text on `blue-50` surface | Informational — signals "you can act in either direction" |
| `disabled` | `zinc-400` text on `zinc-50` surface | Visually inactive without invisible — Pitfall 11 UX side |

Phase 5 must use these specific hues and contrast-tested combinations or amend this contract. The reservation prevents Phase 5 from drifting into mixed semantic systems (e.g. red-for-attention vs amber-for-attention).

**Contrast minimums:** all text/background pairs declared above meet WCAG AA (4.5:1) at 14 px. Body text (`zinc-900` on `zinc-50`) measures 17:1.

### Table component (`ui/src/lib/Table.svelte`)

- **Props:** `{ containers: Container[] }` where `Container` is imported from `./types` (the tygo-generated `ui/src/lib/types.d.ts`).
- **Columns (six, in order):** `container`, `image:tag`, `current digest`, `available digest`, `previous digest`, `status`, `actions`. *(Note: the brief lists `status` and `actions` as separate columns — total six header cells follow the six-name slug from the phase intent: `container | image:tag | current digest | available digest | previous digest | status | actions`. Phase 5 lays out cell content; Phase 1 ships the header bar only.)*
- **Header row:** `bg-zinc-100 border-b border-zinc-200`. Each `<th>` uses `px-4 py-2 text-left text-sm font-semibold text-zinc-700`. Mono columns get a class hook (`font-mono text-xs`) but no text in Phase 1.
- **Empty state row:** when `containers.length === 0`, render a single `<tr>` whose `<td>` spans all 7 columns with `colspan="7"`, contains the empty-state copy, and uses `px-4 py-8 text-center text-sm text-zinc-500 italic`.
- **Table wrapper:** `<table class="w-full border border-zinc-200 rounded-md overflow-hidden">`. Rounded corners + outer border give the table a card-like frame against the `zinc-50` surface.

### Keyboard / focus

No focusable elements in Phase 1 (no buttons, no inputs). The focus-ring contract above is locked for Phase 5 to inherit.

**Voice:** technical, present-tense, second-person where applicable ("Label a service..."), no marketing tone. Field engineers are experts; do not over-explain. Inline `<code>` styling for any label / file path / shell command. No emoji.

**Destructive action policy:** Phase 1 has zero destructive actions. Phase 5 introduces Update / Rollback / Force-pull; the `flutter`/`weston` flicker warning is the only confirmation flow planned (per Pitfall 5 and UX-01). Update / Rollback on non-display services execute immediately without a confirmation modal — disable-on-click + toast on completion (UI-07) is the safety affordance, not a confirm dialog.

## Asset & Cache Contract (Phase 1 → Phase 5 hook)

Phase 5 owns the in-place upgrade test (UI-10, Pitfall 8). Phase 1 must ship the cache strategy so Phase 5 inherits a working baseline:

| Path | `Cache-Control` | Rationale |
|------|----------------|-----------|
| `/` and `/index.html` | `no-cache` | Always re-fetch so post-upgrade asset references are current |
| `/assets/*` (Vite-hashed) | `public, max-age=31536000, immutable` | Hashed filenames are content-addressed; safe to cache forever |
| `/api/*` | `no-store` | State responses must never be cached |
| `/healthz` | `no-store` | Liveness probe must never be cached |

**MIME registration:** the Go static handler registers `.js → application/javascript; charset=utf-8`, `.css → text/css; charset=utf-8`, `.svg → image/svg+xml`, `.json → application/json; charset=utf-8` via `mime.AddExtensionType` at boot. Strict no-fallback on `/assets/*` (404 on miss, never `index.html`) — Pitfall 8 prevention.

This is not a "visual" contract clause in the strict sense, but it is load-bearing for Phase 5's visual delivery and the in-place upgrade test, so it lives here as the single source of truth.

## Acceptance Checks (Phase 1 visual sanity)

These are the visual checks the Phase 1 Playwright smoke test (`e2e/tests/smoke.spec.ts`) should be able to assert without ambiguity:

- [ ] Page background is `rgb(250, 250, 250)` (`zinc-50`)
- [ ] Header bar has visible bottom border (`border-zinc-200`, 1 px)
- [ ] App name `hmi-update` is rendered as `<h1>` at 24 px, weight 600
- [ ] Table renders with exactly seven `<th>` cells with the slugs declared above, in order
- [ ] Empty-state cell uses `colspan="7"`, is centered, italic, `text-zinc-500`
- [ ] Empty-state heading and body copy match this contract verbatim
- [ ] No console errors on initial load
- [ ] `/assets/*.js` returns `Content-Type: application/javascript; charset=utf-8`
- [ ] `/assets/*` returns `Cache-Control: public, max-age=31536000, immutable`
- [ ] `/` returns `Cache-Control: no-cache`
- [ ] At 1024 px viewport width, no horizontal scrollbar appears

The first three header columns (`container`, `image:tag`, `current digest`) must remain visible without truncation at 1024 px (`max-w-screen-xl` is 1280 px, so the table content + 24 px gutters fit comfortably with ~280 px of horizontal slack).
