1. **Dockerfile** — same three-stage shape Phase 1 ships, but production-hardened: explicit base-image digest pin (captured at execute time), version-injection ldflags consuming `ARG VERSION / ARG SHA / ARG BUILT_AT`, `STOPSIGNAL SIGTERM` for clean shutdown, OCI image labels (`org.opencontainers.image.source` etc.), `USER 65532:65532` explicit.

    ```makefile
    .PHONY: build ui types check-types test e2e e2e-cron-fast e2e-debug image image-debug image-prod clean all test-sigkill

    # ... existing targets unchanged ...

    # Production image build. Stamps VERSION/SHA/BUILT_AT into the binary via
    # ldflags so /api/state (or the boot slog line) can identify the running
    # image version. Default VERSION is "dev" — CI / release workflow MUST
    # set VERSION explicitly (e.g. VERSION=v0.7.0 make image-prod).
    #
    # SHA is git short-sha at build time. BUILT_AT is RFC3339 UTC. Both are
    # derived inside the recipe so a no-arg invocation still produces a
    # plausibly-tagged image.
    #
    # The resulting image is hmi-update:phase7-baseline by default (overrideable
    # via IMAGE_TAG). Phase 8's CI pipeline will tag with semver / sha-short /
    # latest via docker/metadata-action — Phase 7's local target is the
    # measurement gate, not the publish gate.
    IMAGE_TAG ?= hmi-update:phase7-baseline
    VERSION   ?= dev
    SHA       ?= $(shell git rev-parse --short HEAD 2>/dev/null || echo "unknown")
    BUILT_AT  ?= $(shell date -u +%Y-%m-%dT%H:%M:%SZ)

    image-prod:
    \tdocker build \
    \t  --build-arg VERSION=$(VERSION) \
    \t  --build-arg SHA=$(SHA) \
    \t  --build-arg BUILT_AT=$(BUILT_AT) \
    \t  -t $(IMAGE_TAG) .
    \t@echo "[image-prod] built $(IMAGE_TAG) (version=$(VERSION) sha=$(SHA) builtAt=$(BUILT_AT))"
    \t@docker image inspect $(IMAGE_TAG) --format 'image size: {{.Size}} bytes' || true
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Build context → image | .dockerignore controls what host paths land in the image. Source-of-truth is the .dockerignore allow-list. |
| ARG values → binary | VERSION/SHA/BUILT_AT flow from Make → docker build --build-arg → ldflags -X → main.version. Untrusted callers (CI) set them; the binary trusts them as informational only (never used for security decisions). |
| Distroless base → final image | gcr.io/distroless/static-debian12:nonroot is the trust root for /etc/ssl/certs/ca-certificates.crt, /etc/passwd nonroot entry, and tzdata. Pinning the digest in a Dockerfile comment is the audit trail; the FROM tag is the human-readable handle. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|

| T-07-01-02 | T | Stale internal/api/dist baked into image | accept | The Dockerfile rebuilds the bundle in stage 1 inside the image graph; .dockerignore explicitly does NOT exclude internal/api/dist (see comment) so local pre-builds also land. Either path produces a fresh bundle. |
